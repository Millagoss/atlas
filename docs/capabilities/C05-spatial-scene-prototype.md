# Engineering Brief

**ID:** C05

**Title:** Spatial Scene Prototype

**Status:** Planned

---

# Objective

Implement the first complete spatial pipeline using deterministic data.

The goal is to prove that Atlas can transform an `ImageAsset` into a renderable `SpatialScene` without relying on AI.

This capability exists to validate the architecture, the scene builder, and the viewer integration independently of machine learning.

---

# Background

Atlas should be able to render a spatial scene regardless of where depth information originates.

For this capability, depth will be generated by a deterministic mock provider.

This creates a stable foundation for future AI providers.

---

# Requirements

Implement a `DepthProvider` abstraction.

Provide a default implementation:

- `MockDepthProvider`

The provider should generate predictable depth information suitable for testing.

Do not attempt to create realistic depth.

The output only needs to exercise the complete pipeline.

---

## SpatialScene Builder

Implement a builder that transforms:

ImageAsset
↓

DepthAsset
↓

SpatialScene

The SpatialScene should become the canonical scene representation used by the viewer.

Design it to support future expansion but avoid unnecessary complexity.

---

## Sandbox Integration

Update the Sandbox.

After image ingestion:

- Execute the mock depth provider.
- Generate a SpatialScene.
- Display a summary of the SpatialScene.

The summary may include:

- node count
- dimensions
- generated metadata
- processing time

No 3D rendering is required yet.

- `DepthProvider` — strategy interface (`name`, `generate(image): Promise<DepthAsset>`).

**`@atlas/scene-engine` (`src/builder.ts`)**

## Architecture Decisions

### Provider/builder as injectable strategies

Both `DepthProvider` and `SpatialSceneBuilder` are interfaces behind factory functions. The pipeline stages are closures over the injected instances so:

- Tests can inject fakes returning deterministic data without needing real models.
- Future AI providers (ONNX, cloud) can replace the mock by swapping the factory at the composition root (`AtlasServicesProvider`) without touching pipeline logic.
- The builder interface supports future enrichment (subdivided geometry, reprojected UVs) without changing the stage.

### Deterministic mock depth

`MockDepthProvider` uses a simple string hash of the source image id to derive pseudo-statistics. Two calls with the same id produce identical depth metadata. This makes integration tests stable and the pipeline fully deterministic for any fixed input id. The mock deliberately produces non-realistic values — it only needs to exercise the pipeline plumbing.

### Scene graph stays minimal

The `DefaultSceneBuilder` creates exactly three nodes (root + 2 children). This satisfies the "avoid unnecessary complexity" requirement while providing a real scene-graph root consumers can traverse. Future builders can add subdivision, material references, or mesh data through the same interface — `SpatialSceneNode` is an open tree by contract.

## Future Considerations

- Move the ingestion pipeline out of the web app into a standalone config package.
- Add `RuntimeScene` (camera + static geometry optimised for the viewer).
